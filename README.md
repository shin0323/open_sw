# 오픈소스 sw 개론 과제 2

## top
- 윈도우의 작업 관리자처럼 시스템의 상태와 프로세스 현황을 실시간으로 갱신하며 보여주고 서버의 부하를 파악할 때 가장 먼저 사용하는 명령어이다.
- 출력 필드 설명
  1. uptime : 시스템이 켜져 있던 시간
  2. users : 현재 접속 중인 사용자 수
  3. load average : 1분, 5분, 15분간의 평균 부하량
  4. tasks : 전체 프로세스 수
  5. %cpu(s) : us(사용자), sy(시스템), id(유휴-idle), wa(io 대기) 등 cpu 사용 내역
  6. mib men.swap : 물리 메모리와 가상 메모리의 사용량/여유량

- 프로세스 리스트 영역
  1. pr, ni : 우선 순위 관련 값
  2. virt, res, shr : 메모리 사용량 (가상, 물리, 공유 메모리)
  3. %cpu, %nem : cpu 및 메모리 점유율

- 단축키
  1. P : cpu 사용률 순 정렬
  2. M : 메모리 사용률 순 정렬
  3. T : 실행 시간 순 정렬
  4. k : 프로세스 종료
  5. c : 명령어의 전체 경로 표시/숨김
  6. i : cpu 코어별 사용량을 펼쳐서 보기
  7. q :  top 종료

## ps
- 현재 실행 중인 프로세스의 목록을 보여주는 명령어이다. top과 달리 실시간으로 갱신되지 않고 명령어를 입력하는 순간의 정적인 상태를 보여준다.
- 주요 옵션
  1. ps -ef : 가장 표준적이고 깔끔한 상태\
     -e : 모든 프로세스를 표시하는 옵션\
     -f : 풀 포맷으로 표시 옵션
  3. ps aux : cpu와 메모리 점유율 볼때 유리
     a : 다른 사용자의 프로세스도 표시\
     u : 사용자 중심의 포맷\
     x : 터미널 제어 없이 실행되는 프로세스(데몬 등)도 표시
     
- 출력 필드 설명
  1. uid : 프로세스를 실행한 사용자 id
  2. pid : 프로세스 고유 번호
  3. ppid : 부모 프로세스의 id
  4. c : cpu 사용률
  5. stime : 프로세스가 시작된 시간
  6. tty : 프로세스가 연결된 터미널
  7. time : 총 cpu 사용 시간
  8. cmd : 실행된 명령어 및 인수

## jobs
- 현재 쉘 세션에서 실행된 백그라운드 작업의 목록을 보여주고 터미널 하나에서 여러 작업을 동시에 돌릴 때 유용하다.
  
- background : 명령 뒤에 &를 붙여 실행하거나 실행 중 ctrl+z로 멈춘뒤 bg 명령을 내린 상태
  
- jobs 출력
  ~~~vim
  [1]- Runing    ./script.sh &
  [2]+ Stopped    vim text.txt
  ~~~
  - [1], [2] : 작업번호
  -  ```+``` : 현재 작업, fg 입력시 가장 먼저 불러오짐
  -  ```-``` : 이전 직업
  -  Running / Stopped : 현재 상태
- 관련 명령어
    - fg %1 : 작업 번호 1번을 포그라운드(화면 앞)로 가져옴
    - bg %2 : 멈춰있는 2번 작업을 백그라운드에서 계속 실행 시킴
    - ctrl ```+``` Z : 현재 실행 중인 포그라운드 작업을 일시 정지시키고 쉘로 빠져나옴

## kill
- 단순히 프로세스를 종료시키는 것이 아닌 프로세스에 특정 신호를 전송하는 명령어이고 가장 많이 쓰는 신호가 종료 신호이다.

- 기본 문법
```vim
kill [옵션/신호] [pid/ %job_id]
```

- 주요 시그널 옵션 종류
  
| 시그널 번호 | 이름 | 설명 |
|:-----:|:---------:|:-----------------------------------------------------------:|
| -15 | SIGTERM | 정상 종료 요청. 프로세스가 하던 작업을 마무리하고 종료할 기회를 줌 |
| -9 | SIGKILL | 강제 종료. 프로세스의 의사를 묻지 않고 커널이 즉시 삭제함. 데이터 손실 위험이 있음 |
| -2 | SIGINT | 인터럽트. 키보드 Ctrl + c를 누른 것과 같음 |
| -1 | SIGHUP | 행업. 보통 설정 파일을 변경 후 프로세스 재시작 없이 적용할 때 사용 |
